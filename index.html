Turn YouTube videos into

TubeFold is a native Mac app that summarizes long videos with the Claude or ChatGPT subscription you already pay for — through Claude Code or Codex. Every note is a plain file on your Mac. No API keys, no account, no TubeFold servers.

How it works

Five steps, one of them yours

Paste a link — or click the Chrome extension while watching. TubeFold does the rest and files the note in your library.

01YouTube URLpasted in the app, or sent with one click from the browser extension
02Transcriptfetched in the video's original spoken language, with the metadata
03Prompta plain-text template you can read and edit wraps the transcript
04Your CLIclaude or codex runs it on your own subscription — no API key involved
05Markdown notestructured summary + metadata front matter, saved to your library as a file

The output

What a note looks like

Structured, sourced, and yours — ready for Obsidian, a docs folder, or anywhere plain text lives. The metadata front matter is generated by the pipeline, never invented by the model.

Why TubeFold

Why another YouTube summarizer?

Because the existing ones charge you a second AI subscription and keep your library on their servers. TubeFold inverts both.

Your subscription, not an API key

TubeFold drives the official claude or codex CLI already signed in on your Mac. No developer console, no per-token billing, no credentials handed to us — summaries cost nothing beyond the plan you have.

Files, not a dashboard

Notes are Markdown with YAML front matter, stored on your disk. They open in Obsidian, they're greppable, and they'd outlive TubeFold itself. There is no cloud copy.

No middleman

There is no TubeFold server. Your transcript goes to exactly one AI company — the one you already trust with your subscription. The full network story is verifiable in the source.

Already pasting transcripts into Claude by hand? That's this pipeline, done manually. TubeFold adds the transcript fetching, a tested prompt, consistent structure, a deduplicated library, one-click capture from the browser, PDF export — and keeps everything.

Open source

Small enough to actually read

TubeFold is MIT-licensed: a SwiftUI app supervising an embedded Python backend, with a pipeline you can trace file by file. The prompt is a Markdown template you can edit. A provider is just an executable that takes a prompt file and writes Markdown back — adding another AI backend, including a local model, is a small shell script. If privacy matters to you, don't take our word for it: read the code.

FAQ

Questions

Why not just paste the transcript into Claude or ChatGPT?
That works — it's the manual version of exactly this pipeline. TubeFold adds automatic transcript fetching (with original-language selection), a consistent tested prompt, metadata front matter, sensible filenames, a deduplicated library, browser one-click capture, and PDF export. If you do it more than a couple of times a week, the automation pays for itself.
Do I need a paid AI subscription?
You need a working claude or codex CLI signed into your account. Summaries run on your plan's quota — TubeFold shows per-provider token usage in Settings so you can see what it consumes.
Is it really local? Does my transcript leave my Mac?
The app, backend, library, and files are local. The AI inference is not — the transcript goes to Anthropic or OpenAI through your own CLI, the same as pasting it into their chat yourself. TubeFold adds no servers of its own, and the complete list of network connections is published above.
Why does it require Claude Code or Codex?
That's how TubeFold uses your existing subscription without ever touching your credentials: the official CLIs handle authentication themselves. It's the trade that makes "no API keys, no second subscription" possible — and if you already have either CLI installed, setup is three minutes.
Why is it macOS-only? Why macOS 26?
One platform, done natively — menu bar, notarization, auto-update, Homebrew. macOS 26 is where the app is built today; lowering the requirement is under evaluation. The Python pipeline underneath is MIT-licensed if you want to run it elsewhere.
Is publishing to Telegraph private?
No — Telegraph pages are public by design, and Telegraph doesn't allow deleting them (only editing). Publish is strictly opt-in, one note at a time, and TubeFold reuses the same page if you publish the same note again.
What happens when the provider CLIs change?
It has already happened once (Codex changed its JSON output format) and TubeFold handles both versions. The provider layer is deliberately thin, and Sparkle auto-update means fixes reach you in days, not release cycles.
Why trust a new open-source project?
Don't trust — read. The privacy-relevant code is a handful of small Python files with no web framework. And because your notes are plain Markdown on your disk, the exit cost is zero: if TubeFold disappeared tomorrow, your notes wouldn't.
